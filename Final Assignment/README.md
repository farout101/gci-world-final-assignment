# GCI World — Final Assignment (summary)

Deadline
-------

- Submission deadline: **2026-06-19 11:00 UTC**. No late submissions accepted.

Submission Portal
-----------------

- Submit via Omnicampus: https://edu.omnicamp.us/courses/137/reports/106/

Required files
--------------

- Slides: PDF, up to **15 slides** total (title, TOC, dividers, references included). File size ≤ 10 MB.
- Notebook: `.ipynb` or `.py` (or `.zip` if multiple files) with reproducible analysis.
- List of References: enter URLs / citations in Omnicampus and include on slides. If you used generative AI, include the chat URL.

File naming
-----------

- Use your Omnicampus account name as filenames: `OmnicampusName.pdf`, `OmnicampusName.ipynb` (or `.py` / `.zip`).

Core requirements (what will be graded)
------------------------------------

- Produce a data-driven business proposal backed by EDA and at least **one ML/statistical model**.
- On-slide, explicitly state **model name**, **evaluation metric**, and the **numeric score**.
- Explain how model insights inform the business proposal (quantified impact: revenue lift, cost savings, risk reduction, etc.).
- Use multiple clear visualizations; make slides understandable to non-ML executives while still justifying technical choices.
- Cite all external data, methods, figures, or text (aim for <40% direct quotations).

Collaboration & academic integrity
---------------------------------

- Discussing with peers is allowed, but copying others’ work is plagiarism and will be penalized.
- If you use any generative AI, include the chat URL in the references and avoid copying outputs verbatim as your own.

Resubmission & late policy
--------------------------

- You may resubmit multiple times before the deadline; only the latest submission is graded.
- No late additions to references or notebooks after the deadline.

Suggested workflow (brief)
--------------------------

1. Market analysis & assumptions (external data, cited).
2. EDA and visualizations — form hypotheses.
3. Define the business problem and target variable with rationale.
4. Feature engineering and experiments; select and evaluate model(s).
5. Translate findings into a business proposal with quantified impact and next steps.

Slide design tips
-----------------

- Be clear in titles and messages; use conceptual diagrams for non-technical audiences.
- Design charts with purpose (colors, legends, units); avoid misleading binning/outliers.

Where to look in this repo
--------------------------
